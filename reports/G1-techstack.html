Selected Techstack — Unchanged from Requirements
Backend
Spring Boot 3.3 + Java 21
VTIT team standard. Java 21 virtual threads handle I/O-heavy notification dispatching. Strong security and data layers. Native Docker image support via Buildpacks.
Alternatives considered: Node.js/Fastify, Python/FastAPI
Frontend
React 18 + Vite 5 + TypeScript 5 + Tailwind CSS 3 + shadcn/ui
Fast HMR, strong type safety catches API contract drift early. TanStack Query for server state. Zustand for client state. pnpm workspace: two apps (admin desktop-first + resident mobile-first) sharing packages/ui.
Alternatives considered: Vue 3/Nuxt, Next.js
Database
PostgreSQL 15
ACID compliant. Native ENUM types for ticket category/status. JSONB for audit log payloads. Partial indexes for SLA queries. Proven at this scale (<10 GB years of operation).
Alternatives considered: MySQL 8, MariaDB
Auth
Spring Security 6 + JWT (JJWT 0.12) + Redis 7
Stateless — no session affinity needed. 15-min access tokens + 7-day refresh tokens. Logout via JWT blocklist in Redis with TTL. BCrypt strength 12. Redis also handles rate-limiting counters.
Alternatives considered: Keycloak, Spring Session
Infrastructure
Docker Compose + Nginx 1.25 + MinIO
Required constraint. Nginx serves React build + proxies /api/*. MinIO self-hosted S3-compatible for ticket photos and contract attachments. Object keys stored in DB; presigned GET URLs issued on demand.
Alternatives considered: AWS S3, Cloudflare R2
Migrations + Notifications
Flyway + Firebase FCM + JavaMailSender + SmsGateway interface
Flyway: SQL-first version-controlled schema. FCM free tier covers 1000 residents. SMTP via JavaMailSender. SMS abstracted behind SmsGateway interface (no-op default; real Vietnamese telco provider injected via @ConditionalOnProperty).
Alternatives considered: Liquibase, AWS SNS

Key Architectural Decisions
  • Modular Monolith over Microservices — 1000 apartments, small team. Distributed services overhead unjustified. Cross-module operations (assign ticket → update contractor rating) benefit from single transaction.
  • Ticket module (not maintenance module) — Feature 2 expanded to general resident request tracking. Single tickets table with category ENUM handles all request types. Contractor assignment constrained at DB level to MAINTENANCE_REPAIR only.
  • DB-level routing enforcement — CHECK constraint assigned_to_contractor_id IS NULL OR category = 'MAINTENANCE_REPAIR' prevents invalid assignments even if service layer is bypassed.
  • SLA per category, stored at INSERT — sla_deadline = created_at + category.sla_hours stored on ticket. Partial index on (sla_deadline) WHERE status NOT IN ('DONE','CANCELLED') keeps hourly scheduler fast.
  • UUID PKs throughout — Sequential integers allow enumeration attacks. No meaningful performance cost at this scale.
  • MinIO: object keys not full URLs — DB stores keys (e.g., tickets/2026/05/uuid.jpg). Presigned GET URLs with 1-hour expiry issued on demand by backend.
  • Notification delivery: fire-and-forget — FCM/SMTP/SMS failures logged WARN, never roll back business transaction. In-app record always created.
  • SMS: pluggable no-op interface — SmsGateway interface; real Vietnamese telco provider injected via @ConditionalOnProperty without code change.
  • SUGGESTION_FEEDBACK special flow — No assignee required. Admin can close directly NEW → DONE. No SLA enforced (best-effort).
