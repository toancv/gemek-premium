Key Decisions Made This Phase
  • All PostgreSQL ENUMs created in V1 Single migration owns all ENUM types to eliminate cross-migration ordering dependencies. — Alternatives: create each ENUM in its module's migration (rejected: fragile ordering).
  • Ticket module replaces maintenance module Single tickets table with 5-category ENUM. Contractor assignment restricted to MAINTENANCE_REPAIR via DB CHECK + service enforcement. — Alternatives: separate tables per request type (rejected: unnecessary complexity).
  • SLA per category, computed at INSERT MAINTENANCE_REPAIR=24h, COMPLAINT=48h, ADMINISTRATIVE/OTHER=72h, SUGGESTION_FEEDBACK=null. sla_deadline stored in DB; scheduler runs indexed range query.
  • Pessimistic lock on amenity booking conflict check PESSIMISTIC_WRITE lock on the time-slot overlap query prevents double-booking under concurrent requests.
  • AuditLog aspect fire-and-forget AuditLog DB write wrapped in try/catch — audit failures are logged WARN but never roll back the business transaction.
  • Manual DTO mapping in TicketService TicketDetailResponse.photos requires FileStorageService.presign() — a side-effectful call. Building DTOs manually keeps all business logic in the service layer.
  • Object keys in DB, presigned URLs on demand MinIO object keys stored in DB; presigned GET URLs (1-hour TTL) generated at response time. URL never expires in DB.
  • JWT blocklist in Redis, all-device logout Logout adds JTI to Redis blocklist. KEYS scan invalidates all refresh tokens for the user simultaneously.
