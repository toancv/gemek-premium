Key Frontend Decisions
  • Access token in Zustand memory only Refresh token in localStorage (gemek_refresh). On page reload, app calls refresh immediately before first render to restore session.
  • Axios 401 interceptor with queued retry On 401: calls POST /api/auth/refresh, re-queues all failed requests, retries with new token. On refresh failure: clears auth + redirects to /login.
  • window.__gemekAuthState bridge Zustand store exposed on window.__gemekAuthState to break circular import between the axios instance and the store.
  • No chart library — Tailwind CSS stat cards Reports use numeric summaries + table breakdowns. No recharts/chart.js. Reduces bundle size.
  • Native date/time inputs <input type="date"> and <input type="time"> for booking forms — no date picker library.
  • RequireRole wrapper for route gating BOARD_MEMBER sees Dashboard, Reports, Apartments only. All other routes redirect to /dashboard.
Known Constraints for Testing Phase

  • FCM web push not wired — notification bell polls unread-count API; no service worker.
  • Build requires pnpm — pnpm install from frontend/, then pnpm --filter admin build.
