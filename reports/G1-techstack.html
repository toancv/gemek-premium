Apartment Management System

Generated: 2026-05-29 · Awaiting CTO Approval · Scale: ~1000 apartments
Selected Techstack
Backend
Spring Boot 3.3 + Java 21
De-facto enterprise standard. Java 21 virtual threads handle I/O-heavy workloads efficiently. Strong security and data layers. Native Docker image support. Aligns with VTIT team convention.
Alternatives considered: Node.js/Fastify, Python/FastAPI
Frontend
React 18 + Vite 5 + TypeScript 5 + Tailwind CSS 3
Fast HMR in dev, strong type safety, large ecosystem. shadcn/ui for accessible component system. TanStack Query for server state, Zustand for client state. Two portals: admin (desktop-first) + resident (mobile-first).
Alternatives considered: Vue 3/Nuxt, Next.js
Database
PostgreSQL 15
ACID compliant. JSONB for audit log old/new values. Excellent Docker support. Partial indexes for soft-status queries. Proven at this scale (<10 GB for years of operation).
Alternatives considered: MySQL 8, MariaDB
Auth
Spring Security 6 + JWT (JJWT 0.12)
Stateless — no session affinity needed. 15-min access tokens + 7-day refresh tokens in Redis. Logout via JWT blocklist in Redis. BCrypt strength 12 for passwords.
Alternatives considered: Keycloak, Spring Session
Infrastructure
Docker Compose + Nginx + Redis 7 + MinIO
Required constraint (single-command startup). Nginx serves React build + proxies /api/*. Redis for JWT blocklist and rate limiting. MinIO (S3-compatible, self-hosted) for maintenance photos and contract attachments — no cloud vendor lock-in.
Alternatives considered: AWS S3, Cloudflare R2
Migrations + Notifications
Flyway + Firebase FCM + JavaMailSender + SMS interface
Flyway: SQL-first, version-controlled schema changes applied on startup. FCM: push notifications to residents (free tier sufficient). JavaMailSender over configurable SMTP. SMS behind pluggable interface — no-op by default, swap in real Vietnamese telco provider without code change.
Alternatives considered: Liquibase, AWS SNS

Key Architectural Decisions
  • Modular Monolith over Microservices — 1000 apartments, small team. Distributed services overhead unjustified. Internal module boundaries clean for future extraction.
  • Single PostgreSQL instance, no schema sharding — Data volume <10 GB for years of operation. Cross-module joins trivial. Shared connection pool.
  • UUID primary keys throughout — Sequential integers allow enumeration attacks. gen_random_uuid() prevents this at no meaningful performance cost.
  • MinIO: store object keys, not full URLs — DB stores keys (e.g., maintenance/2026/05/uuid.jpg). Backend issues presigned GET URLs on demand with 1-hour expiry. URLs never expire in DB.
  • SLA deadline computed at INSERT time — Stored as created_at + category.sla_hours. SLA scheduler runs simple indexed range query without joining categories on every cycle.
  • Notification delivery: fire-and-forget — FCM/SMTP/SMS failures logged at WARN, do not roll back business transaction. In-app record always created.
  • Two separate React apps in pnpm workspace — admin/ (desktop-first) + resident/ (mobile-first). Shared packages/ui component library.
  • SMS gateway: pluggable no-op by default — SmsGateway interface; real Vietnamese telco provider injected via @ConditionalOnProperty without code change.
