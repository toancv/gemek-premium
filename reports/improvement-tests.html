Risk-Prioritized Gap List

Security-sensitive paths ranked highest. Regression of audit-cycle fixes (SEC-01–SEC-22) is highest-impact.

🔴 [GAP-01] FileController / FileStorageService — presign ownership + MIME detection untested  CRITICAL
FileController 36.4% line, FileStorageService 14.7% line. SEC-01 fix (assertPresignAccess) is in code but no test exercises the rejection path — a resident requesting another resident's presign URL. If the ownership check is removed or bypassed, no test catches it. MIME detection via tika-core is almost entirely uncovered; a magic-byte bypass (e.g. polyglot file) would go undetected.
🔴 [GAP-02] TicketServiceImpl — phone strip and presign forbidden branches missing  CRITICAL
57.6% line / 34.8% branch. SEC-03/08 strip phone for TECHNICIAN/BOARD_MEMBER: no test asserts that the response omits phone when caller role is TECHNICIAN. SEC-01 assertPresignAccess 403 path: no test verifies a non-owner receives 403 when requesting a presign URL on another user's ticket attachment.
🟠 [GAP-03] UserController / UserServiceImpl — role promotion to ADMIN entirely untested  HIGH
UserController 29.4% line / 0% branch; UserServiceImpl 31.6% / 12.5%. SEC-04 audit-log on role change not exercised. No test verifies that (a) a non-ADMIN cannot call the role-change endpoint, (b) promoting to ADMIN is logged with actorId, or (c) demoting from ADMIN is refused. Privilege escalation regression would be silent.
🟠 [GAP-04] AmenityServiceImpl — guard rejection branches not tested  HIGH
58.3% line / 31.7% branch. SEC-11 (30-min minimum), SEC-12 (14-day advance window), SEC-22 (past-date) guards exist in code but no test sends a booking that violates each rule and asserts 4xx. Removing any guard would not be caught.
🟠 [GAP-05] AuthServiceImpl — rate-limit exceeded path + null-logout not tested  HIGH
72.6% line / 40% branch. SEC-05: no test drives an IP over the attempt threshold and asserts lockout response. SEC-13: no test sends a logout with missing/null Authorization header and asserts graceful rejection (not NPE). 60% of auth branch paths go untested.
🟠 [GAP-06] ParkingServiceImpl — slot allocation branches largely untested  HIGH
61.6% line / 15.4% branch. Slot assignment, deallocation, conflict detection (double-assignment, over-capacity) not exercised. Guest vehicle approval/rejection paths missing.
🟡 [GAP-07] ContractorServiceImpl — lifecycle transitions untested  MEDIUM
47.5% line / 11.9% branch. PENDING→APPROVED→EXPIRED state transitions not tested. ADMIN-only approval gate not verified by any test.
🟡 [GAP-08] ResidentServiceImpl — apartment assignment guards  MEDIUM
54.0% line / 31.0% branch. Assigning a resident to an already-occupied apartment, or unassigning from a wrong apartment, not tested.
🟡 [GAP-09] Schedulers — 0% branch coverage  MEDIUM
MaintenanceScheduleRunner and ContractExpiryScheduler at 22.7% line / 0% branch. Scheduled jobs that send notifications and mutate statuses are never exercised. Silent cron failures go undetected.
🟡 [GAP-10] AnnouncementServiceImpl — 66.5% line / 27.9% branch  MEDIUM
ADMIN-only publish gate, pagination edge cases, target-audience filtering branches not tested.
🟡 [GAP-11] ApartmentServiceImpl — status transition edges  MEDIUM
70.2% line / 50% branch. AVAILABLE→OCCUPIED→MAINTENANCE transitions partially untested.
⚪ [GAP-12] Mapper classes below 80% (apartment.mapper 55%, resident.mapper 55.6%, contractor.mapper 74.1%)  LOW
MapStruct-generated code — partial coverage because not all DTO field combinations are exercised. Low functional risk.
⚪ [GAP-13] SecurityConfig — CORS rejection path untested (branch 12.5%)  LOW
SEC-21 CORS bean present. No test sends a request from a non-allowlisted origin and asserts rejection.
⚪ [GAP-14] Systemic: overall branch coverage 33.7%  LOW (systemic)
